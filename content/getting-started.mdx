## 项目介绍

Typotech 是一套**排版驱动的技术博客模板**——设计令牌控制全局样式、明暗主题自动跟随系统、MDX 组件混写、零客户端 JS 的文章页。Fork → Vercel，十分钟上线。

### 快速开始

```bash
pnpm install
pnpm dev      # http://localhost:3000
pnpm build    # 生产构建
```

### 项目结构

```text
typotech/
├── app/
│   ├── blog/[slug]/page.tsx     # 文章页 (SSG)
│   ├── blog/page.tsx            # 文章目录

│   ├── layout.tsx               # 根布局 + 字体 + 主题注入
│   ├── globals.css              # 设计令牌 + 排版 + 暗色模式

│   ├── footer.tsx               # 页脚（链接 + 版权）
│   ├── post-list.tsx            # 文章列表（按年份分组）
│   ├── tag-pills.tsx            # 标签药丸
│   └── observable-embed.tsx     # ObservableHQ 交互式图表嵌入
```

## 部署

Fork → Vercel，三步上线：

**1. Fork 仓库**

点击项目 GitHub 页面右上角 **Fork**，将项目复制到你的账户。

**2. 导入 Vercel**

登录 [Vercel](https://vercel.com)，点击 **New Project** → 选择 Fork 的仓库。Vercel 自动识别 Next.js，无需任何配置。

**3. 设置域名**

在 Vercel 项目 Settings → Domains 添加自定义域名，并设置环境变量：

```
NEXT_PUBLIC_SITE_URL = https://你的域名
```

<Caption>每次 push 到 main 分支自动触发部署，SSL 证书、CDN 全部自动处理。</Caption>

## MDX 写作手册

### Frontmatter

每篇文章顶部以 `export const metadata` 声明元数据，TypeScript 语法：

```typescript
export const metadata = {
  title: '文章标题',                // 必填 — 页头 h1 + <title>
  date: '2026.06.22',               // 必填 — YYYY.MM.DD，用于排序
  author: 'yourname',               // 可选 — 显示为 @author
  description: '一句话摘要。',       // 可选 — SEO + RSS + 列表预览
  tags: ['tag1', 'tag2'],           // 可选 — 生成 /tags/[tag] 筛选页
  draft: true,                      // 可选 — true 时跳过生产构建
  image: 'og-image.png',            // 可选 — 社交卡片预览图
};
```

<Caption>每个字段的用途写在行末注释中。`date` 格式必须为 `YYYY.MM.DD`，用 `.` 分隔。`tags` 建议 2—3 个，全小写英文。</Caption>

### 标题层级

正文从 `## ` 开始——页头已渲染 `metadata.title` 为 h1，**正文内不应再出现 `# `**。

| 层级 | 写法 | 效果 |
|------|------|------|
| h2 | `## 章节标题` | 36px / 44px semibold |
| h3 | `### 小节标题` | 28px / 36px semibold |
| h4 | `#### 微型标题` | 22px / 30px semibold |

<Caption>写法：`##` 后留一个空格再接文字。h2 自动带有 `#` 锚点链接，hover 可见。</Caption>

### 图片

#### 无说明图的写法

本地图片放入 `public/images/` 后，**不带路径前缀**直接引用：

```markdown
![架构图](architecture.svg)
```

实际渲染效果——图片自动撑满内容宽度、圆角、懒加载：

![架构图](architecture.svg)

外部图片直接粘贴完整 URL。

#### 带说明的图（Figure）

需要说明文字时用 `<figure>` + `<FigCaption>`——专用图注组件，居中、与 caption 同款 14px / 24px：

```html
<figure>
  <img src="/images/benchmark.svg" alt="Lighthouse 评分" />
  <FigCaption>图 2：Lighthouse 性能评分——SSG 预渲染获得满分。</FigCaption>
</figure>
```

<figure>
  <img src="/images/benchmark.svg" alt="Lighthouse 评分" />
  <FigCaption>图 2：Lighthouse 性能评分——SSG 预渲染获得满分。</FigCaption>
</figure>

<Small>关键规则：用 `<FigCaption>` 而非原生 `<figcaption>`——MDX-RS 不拦截原生 HTML 标签，`<figcaption>` 会丢失所有样式。`<FigCaption>` 走 MDX 组件映射，居中、`mt-3`。图注和表注均用此组件。</Small>

### 数学公式

KaTeX 渲染，支持 LaTeX 语法。

#### 行内公式——`<InlineMath>` 组件，反斜杠需双写 `\\`

```html
当 <InlineMath>{`n \\to \\infty`}</InlineMath> 时级数收敛。
```

当 <InlineMath>{`n \\to \\infty`}</InlineMath> 时级数收敛。

#### 块级公式——`<BlockMath>` 组件，独立居中

```html
<BlockMath>{`\\sum_{k=1}^{n} k = \\frac{n(n+1)}{2}`}</BlockMath>
```

<BlockMath>{`\\sum_{k=1}^{n} k = \\frac{n(n+1)}{2}`}</BlockMath>

<Small>关键规则：反斜杠在 JSX 模板字符串 `` {`...`} `` 内必须双写 `\\`，否则 MDX-RS 将其当作 Unicode 转义序列（`\s`、`\f`、`\i` 等）导致编译错误。</Small>

### 引用块

写法：每行以 `> ` 开头。段落间用 `>` 空行分隔：

```markdown
> 第一段引用文字。
>
> 第二段引用文字。
```

>

## 站点配置

```typescript
export const site = {
  name: 'Typotech',
  description: '排版驱动的技术博客模板。',
  url: 'https://typotech.dev',
  intro: '首页简介（2—3 句）。',
  banner: '/images/banner.svg',
};

export const theme = { /* 明暗主题色 */ };
export const footerLinks = [ /* 页脚链接 */ ];
export const nav = { /* 导航栏 */ };
```

### 自定义 Banner

1. 准备一张横幅图片（建议 1440×480）
2. 放入 `public/images/`
3. 更新 `site.banner` 路径，`pnpm build` 后生效
